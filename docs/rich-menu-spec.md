# リッチメニュー 設計仕様

`予定お知らせ君` のリッチメニュー（LINE画面下部の常設操作パネル）の設計書。

- コンセプト画像: [`rich-menu-concept.svg`](rich-menu-concept.svg)
- サイズ: **2500 × 1686 px**（LINE Messaging API リッチメニュー Large）
- 分割: **3列 × 2段 = 6ボタン**

## レイアウト

|     | 列1 (0-833) | 列2 (833-1666) | 列3 (1666-2500) |
|-----|---|---|---|
| **上段** (0-843) | 📅 新規登録 | 🚩 目標設定 | ⏳ 目標確認 |
| **下段** (843-1686) | ☀️ 今日の予定 | 🌙 明日の予定 | 💬 ヘルプ |

- **上段** = 書き込み操作（タスクを作る・目標を立てる・進捗を見る）
- **下段** = 閲覧操作（予定を確認する・使い方を見る）
- **テーマ**: 上段は朝焼けの暖色、下段は夕暮れの寒色。朝夜通知ボットの世界観と合わせる

## 各ボタンの挙動（実装時の参考）

| ボタン | LINEに送信するテキスト | ボット側の処理 |
|---|---|---|
| 新規登録 | （任意） | 「タスク名を入力してください」と案内 → 既存の対話フロー |
| 目標設定 | `目標設定` | 既存の `showGoalSelection()` を発火 |
| 目標確認 | `目標確認` | **新規実装** 現在設定中の `【目標】` 予定を一覧表示 |
| 今日の予定 | `今日の予定` | **新規実装** 当日分の `CalendarApp.getEventsForDay` を整形して返信 |
| 明日の予定 | `明日の予定` | **新規実装** `notifyNight` の予定リスト部分を流用 |
| ヘルプ | `ヘルプ` | **新規実装** 使い方の要約を返信 |

## 画像生成AIへ渡すプロンプト（Nano Banana / ChatGPT用）

添付のコンセプトSVGをそのまま参照画像として渡し、以下の指示を添える想定。

```
この画像はLINEリッチメニューの設計コンセプトです。同じレイアウト・同じ分割位置・同じアイコンの意味合いを保ったまま、以下の条件で高品質なイラストに仕上げ直してください。

- サイズ: 2500 × 1686 px
- 3列 × 2段 のグリッド構造を厳密に維持
- 上段は朝焼け（クリーム〜アプリコット〜淡いオレンジのグラデーション）
- 下段は夕暮れ（ネイビーブルー〜深い紺のグラデーション）
- 各セルに描かれているアイコン（カレンダー+プラス / 旗 / 砂時計 / 「今」バッジ付きリスト / 「明」バッジ付きリスト / クエスチョンマークの吹き出し）と、その下のラベル文字は必ず残す
- 文字: 「新規登録」「目標設定」「目標確認」「今日の予定」「明日の予定」「ヘルプ」は日本語のまま、視認性重視の太字。英語サブラベル (CREATE / GOAL / PROGRESS / TODAY / TOMORROW / HELP) も残す
- テイスト: フラットデザイン寄りの温かみのあるイラスト。手描き感を少し残す
- セル間の区切りは残すが、主張しすぎない細い白線で
- 全体の彩度は中庸〜やや明るめ。スマホの小画面でも各ボタンが識別できる明瞭なコントラストを確保
```

## 画像ができたあとの流れ

1. 生成された PNG を `assets/rich-menu.png` として保存
2. LINE Messaging API の `/v2/bot/richmenu` エンドポイントへ POST（上記JSON） → `richMenuId` 取得
3. `/v2/bot/richmenu/{richMenuId}/content` へ画像を POST
4. `/v2/bot/user/all/richmenu/{richMenuId}` でデフォルトリッチメニューに設定
5. コード側に `新規登録` / `今日の予定` / `明日の予定` / `目標確認` / `ヘルプ` の各コマンド処理を追加
